# 《野孩子 1.0》遊戲設計文檔 (GDD)

## 卷一：產品概述

### 1.1 遊戲定位

《野孩子 1.0》是一款以日本鄉野怪談為世界觀、支援 1-4 人區域網合作的**回合制箱庭探索 + 卡牌對峙 Roguelike** 網頁遊戲。

玩家扮演誤入「黃昏村」的野孩子們。白天，在精緻的微縮村落地圖上合作探索、解謎、與村民建立羈絆、收集具有記憶的奇物。夜晚，你們必須並肩坐上牌桌，與村子裡的「那個東西」進行一場關乎規則理解與記憶碎片的卡牌對峙。

### 1.2 核心體驗支柱

1. **箱庭協作探索**：在可互動的微縮村落地圖上，與朋友分工合作。每一次點擊都可能觸發事件、發現線索、或驚動不可名狀之物。
2. **七大維度驅動的萬象劇本**：每一局的「追逐者、禁忌、村中氛圍、在場神明、破局條件、地圖佈局、中場轉折」皆由系統隨機組合，從根本上改變玩法與故事。
3. **奇經連鎖與記憶共鳴**：收集散落村中的奇物。當你同時持有兩件互相「記得」的奇物，會觸發「記憶連鎖」，喚醒一段被遺忘的故事，並永久改變本局的規則。
4. **FOR THE KING 式的判定系統**：所有遭遇、解謎、對峙都基於清晰的擲骰檢定。角色天賦、手牌 Tag、隊友協助都會影響成功率。
5. **村民羈絆養成**：與村民的關係是寶貴的跨局資源。從「認識」到「羈絆」，他們將成為你對抗黑暗的關鍵盟友。
6. **壓迫式卡牌對峙**：黑夜降臨，與追逐者在環形牌陣上展開規則對弈。每回合只有一個核心決策：**出哪張牌？換距離還是推進破局進度？**

### 1.3 遊戲標語

> **「天快黑了，這次，你不是一個人。」**

### 1.4 遊戲模式

| 模式 | 說明 |
|------|------|
| **單人劇情模式** | 玩家獨自探索，可操控一名主要角色。聚焦於深度劇情、角色背景故事與大量文字線索。 |
| **多人合作模式** | 2-4 名玩家通過區域網或房間碼加入同一局。共享地圖、時間、任務目標；手牌、物品、關係各自獨立。 |

## 卷二：核心遊戲循環

### 2.1 一局遊戲的完整旅程

一局《野孩子》遊戲，就是一個完整的怪談故事。從踏入黃昏村到天亮離開，全程約 20-30 分鐘。

```
第一階段：黃昏探索（約 12-18 分鐘）
    ↓  （天色漸暗，壓力漸增）
第二階段：深夜牌局（約 5-8 分鐘）
    ↓  （成功破局或燈火熄滅）
第三階段：晨光結算（約 2-3 分鐘）
```

- **第一階段**：在村子裡走動、與村民交談、觸發事件、躲藏求生，收集線索與資源。
- **第二階段**：坐上牌桌，面對追逐者，用手上僅有的幾張牌去兌現白天累積的一切。
- **第三階段**：無論成敗，都會帶走一些「見聞」，讓下一局變得更聰明。

### 2.2 階段流程

**Menu → Setup → Explore → Battle → Victory/Defeat**

1. **MenuPage**：主選單 → 選擇角色（地藏童/狐之乙/巫女姬/怪力童）
2. **SetupPage**：七大維度揭示動畫（逐條顯示本局規則）
3. **ExplorePage**：地圖探索（AP 管理、移動、調查、對話、躲藏）
4. **BattlePage**：環形牌陣對峙（出牌、移動、破局）
5. **VictoryPage / DefeatPage**：結算畫面

## 卷五：卡牌系統

### 5.1 牌面屬性

每張卡牌包含：
- **id**：唯一標識
- **name**：名稱
- **number**：數字（0-10），影響對峙基準勝率
- **tags**：標籤陣列（`守`、`逃`、`咒`、`問`、`真`、`假`、`水`、`夜`）
- **category**：類別（決定卡牌美術風格）
- **effect**：效果文字
- **source**：來源（選填，如追逐者名、村民名、神明名）

### 5.2 卡牌類別

| 類別 | 前綴 | 數量 | 數字範圍 | 說明 |

### 5.3 對峙判定

- 基準勝率基於卡牌數字 vs `chaserThreat`（預設 5）
- 數字 > 威脅者：勝率 70% / 平手 50% / 小於 30%
- Tag 加成：若卡牌 Tag 匹配追逐者的 `weakTags`，勝率 +15%
- Tag 懲罰：若匹配 `fearTags`，勝率 -15%
- 角色加成（怪力童）：+20%
- 奇物加成：視具體效果
- 最終勝率限制在 [5%, 95%]

## 卷八：探索階段（Explore Phase）

### 8.1 AP 系統

- 單人模式：每時段 4 AP
- 多人模式：每時段 6 AP
- 消耗：每項行動（移動、調查、對話、躲藏、購物、休息）各消耗 1 AP
- AP 耗盡 → 天色變暗（午後 → 黃昏 → 薄暮 → 入夜）
- 每次時段過渡重置 AP
- 飯糰道具可回復 1 AP（上限不超過 `apTotal`）

### 8.2 時間系統

| 時段 | AP 上限 | 天色效果 |
|------|---------|----------|
| 午後 (afternoon) | 5 回合 | 晴朗 |
| 黃昏 (dusk) | 5 回合 | 日落西山，開始出現螢火粒子 |
| 薄暮 (twilight) | 4 回合 | 光線昏暗，少數螢火 |
| 入夜 (night) | - | 進入牌局階段 |

### 8.3 地圖系統

- 13 個格子，類型：草地、水田、道路、建築、神社、大樹、山脈、橋樑
- 4×4 grid 佈局
- 連接關係決定可移動範圍
- 部分格子有隱藏物品、地藏像
- 特殊互動：調查、休息、購物、躲藏

### 8.4 躲藏系統（Hide System）

- 可躲藏地形：建築、神社、大樹、迷霧/黑暗中
- 檢測機率公式：Base 30%
  - 迷霧：+10%，黑暗：+20%
  - 建築：-20%
  - 追逐者距離 ≤1：+30%，≤2：+15%
- 屏息時刻（Breath Hold Game）：
  - 按住按鈕保持屏息
  - 心跳指示器加速
  - QTE 提示干擾
  - 進度條顯示持續時間
  - 放手太早/太久 → 失敗
  - 失敗後果：燈火損失 = min(3, 1 + floor(回合/2))，追逐者步數 = min(3, 1 + floor(回合/3))

## 卷九：牌局階段（Battle Phase）

### 9.1 環形牌陣

- 14 個節點（`BOARD_NODE_COUNT = 14`）
- 玩家與追逐者分別在環的兩端
- 節點上的牌：面朝下（需翻開）或面朝上
- 特殊節點：安全區、守護區、障礙物

### 9.2 出牌規則

- 點選手牌選中，再次點擊出牌
- 出牌後觸發牌面效果
- 可沿環形移動（-1 或 +1 步）
- 當玩家與追逐者同節點時，強制觸發對峙（Confrontation）

### 9.3 對峙系統（Confrontation）

- 選擇一張手牌進行擲骰對決
- 擲骰結果：Great Success / Success / Failure / Great Failure
- 成功：擊退追逐者 3 步（`CONFRONTATION_PUSH_BACK`）
- 失敗：失去 2 燈火（`CONFRONTATION_FAIL_LANTERN_LOSS`），逃跑 3 步（`CONFRONTATION_FAIL_FLEE_STEPS`）
- 逃跑階段：shake 動畫 → 「失去 N 燈火」→ 800ms 後自動關閉

### 9.4 燈火系統（Lantern）

- 初始燈火數：~5
- 追逐者每移動一次消耗 1 燈火（除非 `noLanternBurn` 旗標）
- 燈火歸零 → 遊戲失敗
- 燈火可透過效果回復

### 9.5 神明干預

- 每局根據在場神明獲得初始 `divineCharges`（巫女姬：2，其他：1）
- 消耗 1 點發動神明專屬干預效果
- 效果因神明而異（參見 3.4）

### 9.6 破局系統

- 基於當前維度五的破局條件
- 追蹤 `victoryProgress` / `victoryTarget`
- 收集指定 Tag 的牌（如 7 張【守】、3 張【真】等）
- 完成後可宣告破局 → 勝利

### 9.7 追逐者 AI

六種 AI 行為：

| 追逐者 | 移動規則 |
|--------|----------|
| 無面童子 | 朝玩家 1-2 步，被守護節點阻擋 |
| 雨女 | 朝玩家 1-2 步，下雨/迷霧時加速 |
| 山姥 | 朝玩家 1 步，棄牌≥5 時創建障礙 |
| 野篦坊 | 朝玩家 1 步，每兩輪複製牌到環上 |
| 轆轤首 | 頭/身分開各 1-2 步 |
| 亡者 | 朝玩家 1 步，每輪交換手牌 |

### 9.8 Battle Flags 系統

回合級臨時效果旗標：

| 旗標 | 效果 |
|------|------|
| chaserStoppedTurns | 追逐者停止 N 輪 |
| chaserDirectionRandom | 追逐者方向隨機 |
| effectReversalTurns | 牌效果反轉 N 輪 |
| autoConfrontWinCharges | 自動對峙成功 N 次 |
| onlyEscapeCards | 只能出【逃】牌 |
| onlyNightCards | 只能出【夜】牌 |
| noLanternBurn | 追逐者移動不消耗燈火 |
| ghostCardsActive | 鬼牌活躍中 |

## 卷十：遊戲系統

### 10.1 擲骰判定系統（Dice System）

- `rollDice(baseRate, label, bonuses?)` → `DiceResult`
- 結果包含：擲骰值、目標值、加成清單、最終勝率、結果（great_success/success/failure/great_failure）、訊息
- DiceModal 三階段動畫：就緒 → 旋轉 → 揭示

### 10.2 禁忌系統（Taboo System）

- `checkExploreTaboo(context)` / `checkBattleTaboo(context)` → `TabooResult`
- 每次違反：紀錄 `tabooViolations++`
- 違反 ≥3 → 遊戲失敗
- TabooResult 包含：是否違反、罰則（燈火損失/失衡/卡片移除/追逐者加速）、訊息

### 10.3 環境系統（Environment System）

- 環境類型：clear / rain / fog / frost / dark / moonlit
- 影響：AP 消耗、可視範圍、卡牌效果、追逐者行為
- 範例：下雨→【水】牌效果加倍，黑暗→只能出【夜】牌
- 環境持續多回合後自動變化

### 10.4 道具系統（Items）

| ID | 名稱 | 類型 | 效果 |
|----|------|------|------|
| riceball | 飯糰 | 消耗 | 回復 1 AP |
| herb | 藥草 | 消耗 | 回復 5 HP |
| charm | 護身符 | 消耗 | 抵擋一次禁忌 |

### 10.5 商店系統（Shop System）

- 商品隨機生成，基於氛圍、回合數、玩家金錢
- 價格受 `atmosphere.shopPrice` 影響（×0.8 ~ ×1.5）
- 漠然氛圍商店關閉，瘋狂氛圍價格隨機
- 購買時觸發動畫反饋

### 10.6 神明系統（Deity System）

- `getExploreDeityPassive(context)` / `getBattleDeityPassive(context)`
- 探索階段的被動效果（如地藏堂可作安全屋）
- 牌局階段的被動效果（如持【守】牌時追逐者-1步）
- 干預效果（每局 limited 次數）

### 10.7 氛圍系統（Atmosphere System）

- `getAtmosphere(atmosphereId)` → 初始村民羈絆調整值
- 影響：村民行為、商店價格、資訊獲取難度

### 10.8 奇物系統（Relic System）

- `computeRelicEffects(context)` → 活躍奇物效果列表
- `evaluateRelicChain(heldRelicIds)` → 觸發的記憶連鎖
- 奇物效果疊加，鏈鎖效果取代單體效果

### 10.9 追跡者系統（Chaser AI System）

- `computeChaserAI(context)` → 追逐者移動決策
- 基於：追逐者 ID、當前位置、玩家位置、環境、Flag 狀態
- 輸出：移動目標節點
