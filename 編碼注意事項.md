# ⚠️ 編碼警告 — 勿重蹈覆轍

## 問題

2026-06 月，在 zh-TW Windows 上使用 PowerShell 的 `Get-Content -Raw`
讀取 `.ts` / `.tsx` 檔案時，**PowerShell 5.1 預設使用 Big5 (CP950) 編碼**
而非 UTF-8，導致所有中文字元被破壞成 `�` 或亂碼。

受影響檔案：
- `src/store/useGameStore.ts`（最嚴重，1642 行 → 58+ 段 addLog 損毀）
- `src/systems/tabooSystem.ts`（所有禁忌訊息）
- 其他檔案中的註解（較輕微）

## 根本原因

PowerShell 5.1 的 `Get-Content` 沒有 `-Encoding utf8` 參數時會自動
使用系統 ANSI 編碼（在繁體中文 Windows 上即 CP950）。

```powershell
# ❌ 錯誤 — 會損毀 UTF-8 檔案
$content = Get-Content -Raw "file.ts"

# ✅ 正確
$content = Get-Content -Raw "file.ts" -Encoding utf8
```

## 未來操作原則

### 1. 永遠使用 Python 3 進行檔案讀寫

```python
# ✅ 安全
with open('file.ts', 'r', encoding='utf-8') as f:
    content = f.read()
```

### 2. 若用 PowerShell，務必加參數

```powershell
# ✅ 強制 UTF-8
Get-Content -Raw "file.ts" -Encoding utf8
Set-Content -Path "file.ts" -Value $content -Encoding utf8
```

### 3. 不要用沒指定 encoding 的 PowerShell 檔案指令

**禁用**（會自動用 Big5）：
- `Get-Content`
- `Set-Content` / `Out-File`
- `Add-Content`

**使用**（強制 UTF-8）：
- `Get-Content -Encoding utf8`
- `Set-Content -Encoding utf8`
- `Out-File -Encoding utf8`

### 4. 編輯器設定

確保 VSCode / 編輯器的「Auto Guess Encoding」**關閉**，
永遠以 UTF-8 without BOM 儲存。

### 5. 驗證腳本

修改檔案後，執行以下檢查是否有殘留亂碼：

```bash
python3 -c "
with open('檔案路徑', 'r', encoding='utf-8') as f:
    c = f.read()
if '\ufffd' in c:
    print(f'⚠️ 發現 {c.count(chr(0xfffd))} 個損毀字元')
else:
    print('✅ 乾淨')
"
```

## 專案編碼標準

- **所有原始碼**：UTF-8 without BOM
- **行尾**：LF（Unix），不要在 Windows 上轉換
- **不需要任何 BOM 標記**

## 本次修復摘要

| 項目 | 處理方式 |
|------|---------|
| `useGameStore.ts` 中損毀的 addLog | 替換為英文訊息 |
| `tabooSystem.ts` 中文禁忌規則 | 重寫為英文 |
| 所有註解亂碼 | 替換為英文 |
| 中文字串（tag, 維度名稱等） | 保留不變（本身是 UTF-8 編碼，`Get-Content` 不會讀到這些檔案） |
